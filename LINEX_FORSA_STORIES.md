## Part 2: System Stories

### 1. Authentication & Authorization

| ID | Component | System Story | Status |
|----|-----------|-------------|--------|
| SS-AU-01 | Auth | The system shall support user registration with email and secure password storage (bcrypt, 12 rounds) for all four user roles (Owner, Contractor, Engineer, Admin). | ✅ Built |
| SS-AU-02 | Auth | The system shall enforce role-based access control so that each user can only access features permitted by their role, with server-side authorization on all routes. | ✅ Built |
| SS-AU-03 | Auth | The system shall manage sessions using JWT tokens stored in HTTP-only secure cookies with 7-day expiration. | ✅ Built |
| SS-AU-04 | Auth | The system shall provide API endpoints for auth checking (/api/auth/me) and logout (/api/auth/logout). | ✅ Built |

### 2. Project Engine

| ID | Component | System Story | Status |
|----|-----------|-------------|--------|
| SS-PR-01 | Projects | The system shall support three project types (DESIGN_ONLY, CONSTRUCTION_ONLY, DESIGN_AND_CONSTRUCTION) stored as a field on each project with visibility rules based on user role. | ✅ Built |
| SS-PR-02 | Projects | The system shall manage the project lifecycle through statuses (DRAFT → PENDING_REVIEW → PUBLISHED → BIDDING → AWARDED → IN_PROGRESS → COMPLETED → ARCHIVED) with status history tracking. | ✅ Built |
| SS-PR-03 | Projects | The system shall support a bidding window with configurable end date (default 30 days from publication). | ✅ Built (schema) |
| SS-PR-04 | Projects | The system shall store project attachments with file metadata (name, URL, size, MIME type). | ✅ Built (schema) |
